# Документация системы интернационализации

## 1. Обзор

Система многоязычности в этом проекте обеспечивает runtime-переводы для двух языков:
- русский (`ru`) как язык по умолчанию
- английский (`en`) как второй язык

Эта система реализована с помощью кастомного runtime-сервиса переводов, а не через compile-time i18n Angular. Runtime-подход выбран, потому что он позволяет:
- динамически переключать язык в UI без перезагрузки страницы
- сохранять предпочтения языка через `localStorage`
- проще поддерживать и расширять данные переводов с помощью JSON-файлов
- использовать более простую архитектуру для лендинг-приложения, где не требуется извлечение локалей на этапе сборки

## 2. Структура файлов

Система переводов находится в `src/app/core/i18n/`.

### `src/app/core/i18n/language.types.ts`
- Определяет поддерживаемый тип языка:
  - `export type Language = 'ru' | 'en';`
- Централизует объединение языков, чтобы остальная часть приложения оставалась типобезопасной.

### `src/app/core/i18n/translation.service.ts`
- Основной сервис переводов, который обрабатывает состояние языка и данные переводов.
- Загружает JSON-пакеты переводов и предоставляет вспомогательные методы для компонентов.

### `src/app/core/i18n/translate.pipe.ts`
- Предоставляет шаблонный pipe с именем `translate`.
- Преобразует ключи переводов, такие как `HEADER.TITLE`, в локализованный текст.
- Использует TranslationService внутри.

### `src/app/core/i18n/translations/ru.json`
- Содержит все русские ключи переводов и значения.
- Является языком по умолчанию и источником fallback.

### `src/app/core/i18n/translations/en.json`
- Содержит все английские ключи переводов и значения.

## 3. Объяснение основного сервиса

### TranslationService

`TranslationService` предоставлен на уровне root, поэтому один экземпляр разделяется по всему приложению.

#### `currentLanguage$`
- Определен как observable на основе `BehaviorSubject`.
- Излучает текущее значение языка при каждом его изменении.
- Используется UI для синхронизации состояния выбора языка и поддержки реактивных обновлений.

#### `loadTranslations(language)`
- Загружает данные переводов из объекта `translationStore` в памяти.
- Переводы импортируются на этапе сборки из JSON-файлов:
  - `enTranslations` из `translations/en.json`
  - `ruTranslations` из `translations/ru.json`
- Этот метод устанавливает активный объект `translations`, используемый `translate(key)`.
- Если запрошенный язык не найден, используется русский.

#### `switchLanguage(language)`
Метод выполняет следующие шаги:
1. Проверяет, активен ли уже запрошенный язык. Если да, возвращает управление.
2. Вызывает `loadTranslations(language)` для установки активного набора переводов.
3. Сохраняет выбранный язык в `localStorage` под ключом `app-language`.
4. Излучает новый язык через `currentLanguageSubject.next(language)`.

Эта последовательность гарантирует, что выбранный язык сохраняется и все подписчики получают уведомление.

#### `translate(key)`
- Принимает ключ перевода, например `HEADER.TITLE`.
- Ищет ключ в текущем загруженном наборе переводов.
- Если ключ отсутствует в активном языке, он возвращается из русского fallback-набора.
- Если ключ отсутствует полностью, возвращает сам ключ как последний вариант.

Эта логика fallback гарантирует, что отсутствие перевода не ломает UI, а русский остается безопасным вариантом.

## 4. Поток переключения языка

Поток переключения языка работает следующим образом:

1. Пользователь нажимает кнопку языка в `HeaderComponent`.
2. `HeaderComponent.changeLanguage(language)` вызывает `TranslationService.switchLanguage(language)`.
3. `switchLanguage()` загружает новый набор переводов и сохраняет язык в `localStorage`.
4. Сервис испускает новый текущий язык через `currentLanguage$`.
5. Angular-шаблоны, использующие `{{ 'KEY' | translate }}`, переоцениваются, потому что pipe нечистый.
6. Отображаемый текст обновляется сразу без перезагрузки страницы.

Этот поток обеспечивает плавный UI с мгновенной сменой языка.

## 5. Объяснение custom pipe

### `TranslatePipe`
Расположен в `src/app/core/i18n/translate.pipe.ts`.

Как он работает:
- Pipe объявлен с `pure: false`, поэтому Angular вызывает его на каждом цикле обнаружения изменений.
- Он внедряет `TranslationService` и передает ключ в `translationService.translate(key)`.
- Поскольку он нечистый, ему не нужно самостоятельно подписываться на observable языка.
- Pipe переоценивается сразу после изменения состояния сервиса и запуска цикла change detection.

Почему он нечистый:
- Нечистый pipe необходим, потому что результат перевода зависит от внешнего состояния сервиса, а не только от входного ключа.
- Когда язык меняется, один и тот же ключ должен возвращать другой текст.
- `pure: false` обеспечивает мгновенное обновление UI при смене активного языка.

Пример использования в шаблонах:
```html
<h1>{{ 'HEADER.TITLE' | translate }}</h1>
```

Внутренние шаги:
1. Angular оценивает pipe с `'HEADER.TITLE'`.
2. `TranslatePipe.transform('HEADER.TITLE')` вызывает `TranslationService.translate('HEADER.TITLE')`.
3. Проверяется активный набор переводов.
4. Возвращается переведенная строка и рендерится.
5. При смене языка change detection снова вызывает pipe и обновляет строку.

## 6. JSON-файлы переводов

### Структура ключей
- Ключи переводов являются плоскими строками с точечной нотацией, например:
  - `HEADER.TITLE`
  - `HERO.SUBTITLE`
  - `CONTACT.BUTTON`
- Такая структура легко читается и организуется, сохраняя простоту поиска.

### Почему используется плоская структура
- Плоские ключи исключают необходимость обхода вложенных объектов в сервисе переводов.
- Их просто импортировать и поддерживать в JSON-файлах.
- Они обеспечивают единообразие именования в разных разделах.

### Расширение системы
Чтобы добавить новый ключ перевода:
1. Добавьте пару ключ/значение в оба файла `ru.json` и `en.json`.
2. Используйте ключ в шаблонах через pipe `translate`.

Чтобы добавить новый язык:
1. Добавьте новый JSON-файл в `src/app/core/i18n/translations/`, например `de.json`.
2. Обновите `language.types.ts`, добавив новый код языка.
3. Импортируйте новый JSON-пакет в `translation.service.ts`.
4. Добавьте новый пакет в `translationStore`.
5. Опционально добавьте новый элемент управления языком в UI header.

## 7. Текстовое описание диаграммы потока данных

Нажатие кнопки → Сервис → BehaviorSubject → Pipe → Обновление UI

Детальный поток:
- Кнопка языка в заголовке вызывает метод компонента.
- Метод компонента вызывает `TranslationService.switchLanguage()`.
- Сервис обновляет активный набор переводов и испускает новый язык.
- Translate pipe читает текущий пакет и возвращает локализованный текст.
- Angular повторно рендерит изменившиеся шаблоны.

## 8. Расширяемость

Система переводов построена для простого расширения.

### Добавление нового языка
1. Расширьте `Language` в `language.types.ts`.
2. Добавьте новый JSON-файл языка в `translations/`.
3. Импортируйте файл в `translation.service.ts`.
4. Зарегистрируйте его в `translationStore`.
5. Добавьте элементы управления для выбора языка в UI.

### Добавление новых ключей переводов
1. Определите ключ во всех JSON-файлах переводов.
2. Используйте ключ в шаблонах через `{{ 'SECTION.KEY' | translate }}`.

### Масштабирование системы
- Плоская структура ключей позволяет сервису обходиться без вложенного парсинга.
- Сервис остается единственным источником логики переводов.
- Логика fallback `translate` защищает UI от отсутствующих ключей.
- Дополнительные страницы и компоненты могут повторно использовать тот же pipe без дополнительной настройки.

Эта архитектура поддерживает рост без изменения основной реализации переводов.
