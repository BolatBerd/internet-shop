# Обзор проекта

## 1. Описание проекта

Этот проект представляет собой современную Angular-лендинг страницу для бизнеса по изготовлению мебели на заказ. Он разработан как промо-сайт с витриной продукции, описанием услуг, отзывами и формой обратной связи.

Цель:
- Представить мебельный бизнес как аккуратную лендинг-страницу
- Продвигать услуги по изготовлению мебели на заказ и собирать заявки клиентов
- Обеспечить легкий, адаптивный интерфейс просмотра

Технологический стек:
- Angular 21
- SCSS для стилей
- Standalone компоненты Angular
- RxJS для реактивного управления состоянием
- Runtime i18n с JSON-файлами переводов

## 2. Архитектура

Приложение построено по слоистой структуре папок, разделяющей зоны ответственности.

### `core`
- Содержит базовую логику приложения и повторно используемые сервисы.
- Включает модели, общие сервисы и систему runtime i18n.
- Отвечает за поведение приложения в целом, а не за компоновку интерфейса.

### `shared`
- Содержит повторно используемые UI-компоненты, такие как кнопки, карточки и заголовки секций.
- Компоненты в `shared` предназначены для использования на нескольких страницах.
- Помогает сохранять разметку консистентной и избегать дублирования.

### `layout`
- Содержит компоненты оформления сайта, отображающиеся на всей лендинг-странице.
- Включает компоненты `header` и `footer`.
- Отвечает за глобальную навигацию, бренд, переключение языка и содержимое футера.

### `pages`
- Содержит компоненты, специфичные для экрана или секции.
- Страница `home` собирает разделы лендинга.
- Подпапки внутри `home` соответствуют отдельным секциям: `hero`, `advantages`, `categories`, `works`, `about`, `reviews` и `contact-form`.

## 3. Структура компонентов

Приложение собрано из standalone компонентов, что упрощает повторное использование и объявление зависимостей.

### Композиция страницы
- `src/app/pages/home/home.component.ts` импортирует и рендерит основные секции лендинга:
  - `app-header`
  - `app-hero`
  - `app-advantages`
  - `app-categories`
  - `app-works`
  - `app-about`
  - `app-reviews`
  - `app-contact-form`
  - `app-footer`
- `home.component.html` представляет собой линейную композицию этих секций, что делает структуру страницы понятной и удобной для поддержки.

### Повторное использование shared-компонентов
- `app/shared/components/button/button.component.ts` предоставляет стилизованную кнопку, используемую в нескольких секциях.
- `app/shared/components/section-title/section-title.component.ts` стандартизирует заголовки секций и подзаголовки.
- `app/shared/components/card/card.component.ts` используется для рендеринга повторяющихся карточек в разделах `advantages` и `categories`.

### Компоненты layout
- `app/layout/header/header.component.ts` управляет верхней навигацией и переключением языка.
- `app/layout/footer/footer.component.ts` отображает содержимое футера и контактные данные.
- Компоненты layout являются standalone и импортируются напрямую в дерево компонентов страницы.

### Плавающий контактный виджет (улучшенная версия)
- `app/shared/components/floating-contact/floating-contact.component.ts` отвечает за фиксированную кнопку связи и модальный чат.
- Виджет добавляет слой взаимодействия UI и позволяет пользователю выбрать мессенджер, ввести сообщение и перейти к внешнему приложению.
- Контактные параметры теперь хранятся в environment-конфигурации, а не жестко зашиты в компоненте.
- Компонент встроен в `HomeComponent` и работает независимо от основной структуры страниц.

## 4. Поток данных

### Взаимодействие компонентов
- Проект использует преимущественно иерархию компонентов сверху вниз.
- Секции страницы собираются внутри `HomeComponent` и не передают между собой сложное состояние.
- Статический контент задается напрямую в шаблонах или через ключи переводов.

### Сервисы и состояние
- `TranslationService` из `src/app/core/i18n/translation.service.ts` является основным общим сервисом.
- Он управляет текущим языком и данными переводов.
- Компоненты получают локализованный текст через pipe `translate` или методы сервиса.
- Языковое состояние сохраняется в `localStorage`, поэтому предпочтение сохраняется после перезагрузки страницы.

### Реактивное отображение
- `TranslationService.currentLanguage$` доступен как observable.
- Компонент header косвенно подписывается на него через `translate` pipe и переключатель языка.
- При смене языка шаблоны автоматически обновляются через нечистый pipe `translate`.

## 5. Итоги

Этот Angular-проект структурирован так, чтобы UI, состояние и переводы были четко разделены. Слой `core` содержит повторно используемую бизнес-логику, `shared` содержит повторно используемые UI-примитивы, `layout` отвечает за постоянную структуру страницы, а `pages` определяет содержимое секций. Полученная архитектура удобна для поддержки, масштабирования и расширения новыми страницами или языками.
