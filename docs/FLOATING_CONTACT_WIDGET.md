# Плавающий виджет связи

## Описание функции

Плавающий виджет связи добавляет постоянную кнопку в правом нижнем углу экрана. Он поддерживает выбор канала общения и открывает чат-панель, где пользователь может выбрать Telegram, WhatsApp, Instagram, VK, телефонный звонок или SMS. Виджет помогает мгновенно связаться с мебельным бизнесом, не покидая лендинг-страницу.

## Структура компонента

Компонент реализован как standalone компонент:
- `src/app/shared/components/floating-contact/floating-contact.component.ts`
- `src/app/shared/components/floating-contact/floating-contact.component.html`
- `src/app/shared/components/floating-contact/floating-contact.component.scss`

Компонент импортируется в `HomeComponent` и рендерится вместе с остальными секциями лендинга.

## Архитектурные изменения

До рефакторинга контактные данные и имена профилей были жестко закодированы прямо внутри `FloatingContactComponent`.
После рефакторинга все параметры перенесены в environment-конфигурацию (`src/environments/environment.ts`, `src/environments/environment.prod.ts`).
Это разделяет логику UI и данные контактов, упрощает поддержку и позволяет менять контакты без правки компонента.

## Система конфигурации

Контактные данные хранятся в объекте `environment.contacts`.
Файлы окружения различаются для development и production:
- `src/environments/environment.ts` — dev-среда
- `src/environments/environment.prod.ts` — prod-сборка

В компоненте данные читаются через импорт `environment` и используются при построении URL.

## Логика компонента

Компонент импортирует `environment` из `src/environments/environment` и использует значения:
- `environment.contacts.telegram`
- `environment.contacts.whatsapp`
- `environment.contacts.instagram`
- `environment.contacts.vk`
- `environment.contacts.phone`

Метод `buildUrl(channel)` больше не содержит жестко закодированных профилей. Вместо этого он делегирует генерацию ссылок в специальные методы:
- `getTelegramLink()`
- `getWhatsAppLink()`
- `getInstagramLink()`
- `getVKLink()`
- `getPhoneLink()`
- `getSmsLink()`

Каждый метод проверяет значение из environment и возвращает корректную ссылку. При отправке сообщения используется динамически сгенерированный URL без прямого захардкоженного контента.

## Поведение UI

## Поведение UI

- Кнопка всегда видима в нижнем правом углу.
- Для кнопки создана анимация пульсации и волны, чтобы привлечь внимание.
- Нажатие открывает модальный чат-панель с затемненным фоном.
- Пользователь сначала выбирает канал связи, затем вводит имя и сообщение.
- Кнопка отвечает плавным масштабированием при наведении.

## Логика выбора канала

Компонент хранит состояние выбранного канала:
- `selectedChannel`
- `modalOpen`
- `message`
- `name`

Пользователь выбирает один из каналов через карточки. После выбора карточки отображается чат-панель, где можно ввести данные.

## Логика построения URL

Метод `buildUrl(channel)` формирует URL в зависимости от выбранного канала:
- Для Telegram: `https://t.me/<username>?text=message`
- Для WhatsApp: `https://wa.me/<phone>?text=message`
- Для Instagram: `https://instagram.com/<profile>`
- Для VK: `https://vk.com/<profile>`
- Для телефона: `tel:<phone>`
- Для SMS: `sms:<phone>?body=<message>`

Сообщение кодируется с помощью `encodeURIComponent` и объединяется с именем, если оно введено.

## Как работает отправка сообщения

1. Пользователь выбирает канал.
2. Вводит имя и текст сообщения.
3. Нажимает кнопку "Отправить".
4. Компонент строит ссылку через `buildUrl()`.
5. Web-ссылки открываются в новой вкладке, а `tel:`/`sms:` используются через `window.location.href`.
6. После отправки панель закрывается и форма сбрасывается.

## Расширение новыми платформами

Чтобы добавить новый канал связи:
1. Добавить новый объект в массив `channels` в `floating-contact.component.ts`.
2. Задать уникальный `id`, `labelKey` и `color`.
3. Добавить обработку нового `id` в `buildUrl(channel)`.
4. Добавить ключи перевода в `ru.json` и `en.json`, если используется i18n.
5. Возможно, добавить новый SVG-иконку или визуальный стиль для этого канала.

Такой компонент легко масштабируется за счет централизованной логики каналов и единого метода построения URL.
